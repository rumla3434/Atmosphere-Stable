# 大气层1.9.0整合包测试版（更新时间：2025.05.03）

原主页github.com/rumla34作废，改地址为github.com/rumla3434，另一个主页codeberg.org/rumla34不变。

2025.05.03更新测试版的atmosphere-1.9.0，hekate6.2.2，最高支持真实系统20.0.1，最高支持虚拟系统19.0.1

（1）双系统玩家，如果误升级真实系统到20.0.1的，建议更新此包，不会影响虚拟系统19.0.1或以下。

（2）只玩虚拟系统的玩家，只要不更新真实系统到20.0.0或以上，就不需要更新大气层，安心呆在虚拟19.0.1玩游戏。

（3）只玩真实系统的玩家，比如BAN机的，如果误升级真实系统到20.0.1的，也可以用此包将真实系统降级到19.0.1玩破解游戏。

更新Checkpoint.nro--v3.10.0（无需重新制作前端NSP），NX-Activity-Log.nro--v1.5.7（和checkpoint一样必须以前端模式进），更新DBI同文件夹下DBI.nro.ru是俄文版DBI781，更新Lockpick_RCM.bin--v1.9.14。

目前已知问题：虚拟系统最高只能支持19.0.1，强刷虚拟20.0.0必黑屏，等atmosphere和hekate更新虚拟模块才能进。极限超频loader.kip不支持，需要转回普通超频插件才可以正常，教程「分享」普通超频sys-clk可以和极限超频EOS互相转换，sigpatch最高只能支持19.0.1，大气层自动识别fusee.bin未编译不支持sigpatch，暂时屏蔽fusee引导设置。
